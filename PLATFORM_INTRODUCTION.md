# Agent Monitor 플랫폼 소개

## 📋 개요

**Agent Monitor**는 AI 에이전트를 안전하고 효율적으로 관리하고 모니터링하는 엔터프라이즈급 오케스트레이션 플랫폼입니다. 사용자의 승인을 기반으로 한 **Human-in-the-Loop** 방식으로 AI 에이전트가 외부 서비스와 안전하게 상호작용할 수 있도록 지원합니다.

---

## 🎯 핵심 가치 제안

### 1. **안전한 AI 자동화**
- 모든 외부 작업은 사용자 승인 필수
- 에이전트가 독단적으로 결정하지 않음
- 사용자의 규칙과 선호도를 철저히 준수

### 2. **동적 워크플로우 오케스트레이션**
- 복잡한 작업을 여러 에이전트가 협업하여 처리
- 사용자 요청에 따라 실시간으로 워크플로우 생성
- 단계별 진행 상황을 투명하게 추적

### 3. **실시간 모니터링 및 제어**
- WebSocket 기반 실시간 상태 동기화
- 에이전트의 사고 과정(Thinking Mode) 가시화
- 언제든지 일시정지, 재개, 중단 가능

---

## 🚀 핵심 Feature

### 1. **다중 에이전트 관리 시스템**

#### 주요 기능
- **에이전트 생성 및 구성**: 다양한 유형의 전문 에이전트 생성 (문서 처리, 이메일 처리, 리서치 등)
- **에이전트 라이프사이클 관리**: 생성, 시작, 일시정지, 재개, 중단
- **실시간 상태 모니터링**: 각 에이전트의 현재 상태, 작업 진행률, 통계 정보 실시간 확인

#### 기술적 특징
- **Thinking Mode State Machine**: 에이전트의 사고 과정을 4단계로 구조화
  - `idle` → `exploring` → `structuring` → `validating` → `summarizing`
- **이벤트 기반 아키텍처**: 모든 에이전트 이벤트를 중앙에서 관리 및 브로드캐스트

#### 비즈니스 가치
- 여러 작업을 동시에 병렬 처리하여 생산성 향상
- 각 에이전트의 전문성을 활용한 효율적인 작업 분담

---

### 2. **동적 워크플로우 오케스트레이션**

#### 주요 기능
- **자동 워크플로우 생성**: 사용자의 자연어 요청을 분석하여 최적의 워크플로우 자동 생성
- **다단계 작업 처리**: 복잡한 작업을 여러 에이전트가 순차/병렬로 처리
- **사용자 상호작용 지원**: 필요한 시점에 사용자에게 질문하고 입력 받기

#### 작동 예시
```
사용자: "점심메뉴 추천해주고 근처 식당을 예약해줘"

워크플로우:
1. 메뉴 추천 Agent (백그라운드 작업)
   ↓
2. Q&A Agent (사용자에게 메뉴 선택 질문)
   ↓
3. 식당 예약 Agent (백그라운드 작업)
   ↓
4. Q&A Agent (예약 확인 질문)
   ↓
5. 최종 응답
```

#### 기술적 특징
- **Planner Agent**: LLM 기반 워크플로우 계획 수립
- **Task Schema**: 도메인별 작업 로직 정의 및 재사용
- **Conversation State**: 대화 상태를 구조화하여 관리

#### 비즈니스 가치
- 복잡한 업무를 단순한 요청으로 자동화
- 사용자 경험 향상 (한 번의 요청으로 여러 작업 완료)

---

### 3. **승인 기반 워크플로우 (Human-in-the-Loop)**

#### 주요 기능
- **5가지 승인 유형 지원**:
  1. **Proceed**: 단순 승인/거부
  2. **Select Option**: 여러 옵션 중 선택
  3. **Provide Input**: 필수 정보 입력
  4. **Confirm Action**: 위험한 작업 확인
  5. **Review Result**: 결과 검토 및 승인

- **승인 큐 관리**: 모든 승인 요청을 중앙에서 관리
- **우선순위 기반 정렬**: 긴급한 승인 요청을 먼저 처리

#### 기술적 특징
- **MCP 서비스 통합**: 외부 서비스 작업에 대한 승인 정책 자동 적용
- **Ontology 기반 검증**: 사용자 규칙과 금기사항을 자동으로 검증

#### 비즈니스 가치
- **안전성**: 실수나 잘못된 작업 방지
- **투명성**: 모든 작업을 사용자가 사전에 확인
- **컴플라이언스**: 규정 준수 및 감사 추적 가능

---

### 4. **MCP (Model Context Protocol) 서비스 통합**

#### 지원 서비스
- **Notion**: 페이지 읽기/생성/수정
- **Gmail**: 이메일 읽기/작성/발송
- **Slack**: 메시지 읽기/작성/발송
- **Confluence**: 문서 관리

#### 주요 기능
- **플러그인 아키텍처**: 새로운 서비스를 쉽게 추가 가능
- **자동 승인 정책**: 서비스별로 작업 유형에 따라 승인 필요 여부 자동 결정
  - 읽기 작업: 승인 불필요
  - 쓰기/삭제 작업: 승인 필수
- **롤백 지원**: 작업 실패 시 자동 롤백

#### 기술적 특징
- **BaseMCPService**: 모든 MCP 서비스의 공통 인터페이스
- **서비스 레지스트리**: 중앙에서 모든 서비스 관리
- **비동기 처리**: 모든 외부 API 호출을 비동기로 처리

#### 비즈니스 가치
- **통합 관리**: 여러 서비스를 하나의 플랫폼에서 관리
- **확장성**: 새로운 서비스를 빠르게 추가 가능
- **일관성**: 모든 서비스에 동일한 승인 정책 적용

---

### 5. **실시간 모니터링 및 대시보드**

#### 주요 기능
- **실시간 상태 업데이트**: WebSocket을 통한 즉각적인 상태 동기화
- **에이전트 대시보드**: 모든 에이전트의 상태를 한눈에 확인
- **태스크 추적**: 각 태스크의 진행 상황, 관련 티켓, 승인 요청을 상세히 확인
- **통계 및 분석**: 에이전트별 작업 통계, 완료율, 성공률 등

#### 기술적 특징
- **WebSocket 서버**: 양방향 실시간 통신
- **이벤트 브로드캐스팅**: 모든 상태 변경을 즉시 모든 클라이언트에 전송
- **Heartbeat 모니터링**: 연결 상태 자동 감지 및 재연결

#### 비즈니스 가치
- **가시성**: 모든 작업의 진행 상황을 실시간으로 확인
- **신속한 대응**: 문제 발생 시 즉시 인지 및 조치 가능

---

### 6. **Slack 통합 및 자동 태스크 생성**

#### 주요 기능
- **Slack 멘션 자동 감지**: 봇을 멘션하면 자동으로 태스크 생성
- **DM 지원**: 봇에게 직접 메시지를 보내면 태스크 생성
- **자동 할당**: Slack에서 생성된 태스크는 우선순위가 높게 설정되어 자동 할당

#### 작동 방식
```
Slack 사용자가 봇 멘션
    ↓
Webhook으로 이벤트 수신
    ↓
태스크 자동 생성 (source: slack, autoAssign: true)
    ↓
적절한 에이전트에 자동 할당
    ↓
에이전트가 작업 시작
```

#### 기술적 특징
- **서명 검증**: Slack 서명을 검증하여 보안 강화
- **이벤트 필터링**: 필요한 이벤트만 처리

#### 비즈니스 가치
- **접근성**: 익숙한 Slack 인터페이스에서 바로 작업 요청
- **자동화**: 수동 태스크 생성 없이 자동으로 처리

---

### 7. **개인화 및 온톨로지 관리**

#### 주요 기능
- **사용자 선호도 학습**: 대화를 통해 사용자의 선호도, 규칙, 금기사항 자동 추출
- **온톨로지 기반 검증**: 에이전트가 사용자 규칙을 위반하지 않도록 자동 검증
- **개인화 정보 저장**: 선호(preference), 사실(fact), 규칙(rule), 인사이트(insight) 등 체계적 관리

#### 기술적 특징
- **자동 추출**: LLM을 활용하여 대화에서 중요한 정보 자동 추출
- **카테고리 분류**: 정보를 체계적으로 분류하여 저장
- **향후 확장**: Knowledge Base로 임베딩하여 에이전트가 활용

#### 비즈니스 가치
- **맞춤형 경험**: 사용자별로 최적화된 작업 처리
- **일관성**: 사용자의 규칙과 선호도를 일관되게 준수

---

### 8. **태스크 관리 및 오케스트레이션**

#### 주요 기능
- **태스크 생성**: 수동 또는 자동(Slack 등)으로 태스크 생성
- **자동 할당**: LLM 기반으로 최적의 에이전트 자동 선택
- **수동 할당**: 사용자가 직접 에이전트 선택 가능
- **태스크 상세 보기**: 태스크별 티켓, 승인 요청, 진행 상황 상세 확인

#### 기술적 특징
- **Orchestration Service**: LLM을 활용한 지능형 에이전트 선택
- **우선순위 기반 정렬**: 긴급/높은 우선순위 태스크 우선 처리
- **필터링 및 검색**: 상태, 우선순위, 소스별 필터링

#### 비즈니스 가치
- **효율성**: 최적의 에이전트에 작업 할당으로 처리 시간 단축
- **유연성**: 자동/수동 할당 모드 선택 가능

## 🏗️ 기술 아키텍처

### 프론트엔드
- **React 19** + **TypeScript**: 최신 React 기능 활용
- **Tailwind CSS**: 빠른 UI 개발
- **WebSocket**: 실시간 양방향 통신
- **Vite**: 빠른 개발 서버 및 빌드

### 백엔드
- **Python 3.9+**: 비동기 처리 중심
- **FastAPI**: 고성능 API 서버
- **WebSocket Server**: 실시간 통신 서버
- **Pydantic**: 타입 안전한 데이터 검증

### 통신 프로토콜
- **WebSocket (포트 8080)**: 실시간 이벤트 브로드캐스팅
- **HTTP/REST (포트 8000)**: Slack 웹훅 및 API 엔드포인트

### 외부 서비스 통합
- **MCP (Model Context Protocol)**: 표준화된 서비스 통합 인터페이스
- **OAuth2**: Gmail 등 인증 필요 서비스 지원

## 📊 사용 시나리오

### 시나리오 1: 이메일 자동 응답
```
1. Slack에서 "고객 이메일 확인하고 답변 작성해줘" 요청
2. Gmail Agent가 최신 이메일 확인
3. 이메일 내용 분석 후 답변 초안 작성
4. 사용자 승인 요청 (승인 큐에 표시)
5. 사용자 승인 후 자동 발송
```

### 시나리오 2: 문서 작성 및 공유
```
1. "프로젝트 회의록 작성해서 Notion에 올려줘" 요청
2. 회의록 작성 Agent가 문서 생성
3. Notion 페이지 생성 승인 요청
4. 승인 후 Notion에 자동 업로드
5. Slack에 완료 알림 발송
```

### 시나리오 3: 복합 작업 처리
```
1. "오늘 할 일 정리하고 우선순위 매겨서 Slack에 공유해줘"
2. 할 일 수집 Agent가 여러 소스에서 정보 수집
3. 우선순위 분석 Agent가 분석
4. Slack 메시지 작성 및 승인 요청
5. 승인 후 Slack 채널에 공유
```
